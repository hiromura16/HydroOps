# コントリビューションガイド

Hydro-Opsプロジェクトへの貢献をありがとうございます！このドキュメントでは、プロジェクトへの貢献方法を説明します。

## コントリビューションの方法

### バグレポート

バグを発見した場合は、GitHubの[Issues](https://github.com/hiromura16/HydroOps/issues)で報告してください。

バグレポートには以下の情報を含めてください：
- 問題の説明
- 再現手順
- 期待される動作
- 実際の動作
- 環境情報（OS、Dockerバージョンなど）
- エラーログ（該当する場合）

### 機能リクエスト

新機能の提案がある場合は、GitHubの[Issues](https://github.com/hiromura16/HydroOps/issues)で提案してください。

機能リクエストには以下の情報を含めてください：
- 機能の説明
- この機能が解決する問題
- 提案する実装方法（任意）
- 代替案（任意）

### プルリクエスト

1. リポジトリをフォークしてください
2. 新しいブランチを作成してください（`git checkout -b feature/amazing-feature`）
3. 変更をコミットしてください（`git commit -m 'Add some amazing feature'`）
4. ブランチにプッシュしてください（`git push origin feature/amazing-feature`）
5. プルリクエストを作成してください

## コーディング規約

- 既存のコードスタイルに従ってください
- 変更には適切なコメントを追加してください
- ドキュメントを更新してください（該当する場合）
- テストを追加してください（該当する場合）

## コミットメッセージ

コミットメッセージは明確で説明的にしてください：

```
feat: 新機能の追加
fix: バグ修正
docs: ドキュメントの更新
style: コードスタイルの変更（機能に影響しない）
refactor: リファクタリング
test: テストの追加・修正
chore: ビルドプロセスやツールの変更
```

## 質問

質問がある場合は、GitHubの[Issues](https://github.com/hiromura16/HydroOps/issues)で質問してください。
